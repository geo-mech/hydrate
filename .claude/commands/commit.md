---
description: 提交代码（自动升级版本号、更新日志、生成描述）
---

执行以下操作：

1. **读取未提交修改**：运行 `git diff --stat HEAD` 和 `git status`，阅读实际改动内容，用 1-2 句话总结本次修改的核心内容

2. **升级版本号**：读取 pyproject.toml，将最小版本号（z）加 1（如 1.7.1 → 1.7.2）

3. **更新 CHANGELOG.md**：如果上一条目是同一天，合并到该条目中（追加分类内容）；否则在顶部插入新版本条目。格式：
   ```
   ## v新版本 (日期)
   
   ### 分类标题
   - 具体变更
   ```
   根据 `git diff --stat` 和变更内容自动归纳到合适分类（新增/变更/修复/文档/弃用）

4. **同步 ReadMe**：检查 `git diff --stat` 中涉及的子包和 `zmlx/` 核心，如果改动较大，更新 `zmlx/ReadMe.md` 和对应子包的 `ReadMe.md`

5. **生成提交描述**：中文，结构化，基于第 1 步的总结展开

6. `git add -A && git commit`
