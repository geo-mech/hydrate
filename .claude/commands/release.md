---
description: 公开发布新版本，整理合并修改日志
---

执行以下操作：

1. **升级中间版本号**：pyproject.toml 中 y 加 1，z 重置为 0（如 1.6.33 → 1.7.0）

2. **整理 CHANGELOG.md**：将自上次 release 以来的所有增量条目合并为一个发布条目，删除零散的 patch 记录。发布条目应**全面覆盖**以下内容：
   - **新增**：新功能、新模块、新 API
   - **变更**：行为调整、参数重命名、默认值变化
   - **弃用**：标记为废弃的功能及替代方案
   - **修复**：bug 修复
   - **文档**：新增或重大更新的文档
   
   格式：
   ```
   ## v1.7.0 (日期)
   
   综合概述（一段话，概括本次发布的核心变化）。
   
   ### 新增
   - ...
   
   ### 变更
   - ...
   
   ### 弃用
   - ...
   
   ### 修复
   - ...
   
   ### 文档
   - ...
   ```

3. `git add -A && git commit`
